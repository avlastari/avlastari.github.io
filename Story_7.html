Figma is my playground and my workshop

From building “boring” UI components from scratch, to setting up variables with properties and variant swaps, I enjoy creating the fundamental building blocks of design systems.

But I don’t stop there — I also love experimenting with advanced techniques and motion, using Figma to explore how interfaces feel, not just how they look.

Systems

In large teams, a single screen design isn’t enough. Products need patterns, components, and rules that scale across multiple squads and platforms. With Figma, I’ve:

  • Built and maintained component libraries that speed up design work.
  • Defined tokens for colors, typography, and spacing to ensure consistency.
  • Documented usage rules so developers and designers can work seamlessly together.

My focus:

  • Creating core libraries of buttons, inputs, forms, and navigation elements.
  • Setting up variables and properties that make systems scalable and flexible.
  • Documenting component usage so teams can move faster and more consistently.
  • Exploring motion prototypes directly inside Figma for richer experiences.
I’ll admit it: I geek out over clean components and neat documentation. But it’s not about perfectionism — it’s about creating a shared language for teams. Design systems turn design from a series of isolated screens into a cohesive ecosystem, and that’s what makes them powerful.
